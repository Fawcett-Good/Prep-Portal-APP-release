# Prep Portal

Prep Portal is a mobile application designed to help students navigate campus life at Fairmont Preparatory Academy by providing tools such as a campus map, daily schedule viewer, and event information to help students stay organized and informed. Flutter is utilized for Frontend, Spring Boot is utilized for backend. This Wiki page analyzes the entire project based on the source code and directory structure.

## I. Flutter App Section

Tech Stack: Flutter(Dart). Dependencies include `http`, `google_maps_flutter`, `google_sign_in`, `firebase_core`, `firebase_auth`, `image_picker`, `shared_preferences`, `video_player`, `url_launcher`, `flutter_markdown`, etc. Supports Android/iOS.

The following lists and explains each business related file (excluding auto-generated or 3rd-party files).

| File Path | Purpose |
|----------|------|

| **lib/pages/my_page.dart** | "My" Page: Profile management for users; login prompts for guests. |

### 1.3 Flutter Build and Run

   - Local Run: `cd app && flutter pub get && flutter run` (Requires backend and Firebase setup).
   - Release Build: `flutter build apk` or `flutter build ios`. APKs are output to `build/app/outputs/`.
   - Assets: Images are located in `assets/images/`. Launcher icons are generated via `flutter_launcher_icons`.

## 2.3 Database and JPA

- **Data source**: Configure MySQL (url, username, password) in `application.properties`, with the driver being `com.mysql.cj.jdbc.Driver`.
- **All tables use JPA Entity markings**: Every business table's entity class uses `@Entity` and `@Table`. JPA handles schema synchronization at startup (ddl-auto=update). Tables include: `user`, `user_token`, `admin`, `admin_token`, `classroom`, `portal_event`, `schedule_period`, `user_schedule_period`, `welcome_video`, `content_page`.
- **Data Access Method**: Some modules use **MyBatis Mappers** (`User`, `Classroom`, `Event`, `Schedule`, `WelcomeVideo`) for CRUD and complex queries; `user_token`, `admin`, `admin_token`, and `content_page` use **JPA Repositories**. The schema is defined by JPA entities, while the access layer uses both JPA and MyBatis concurrently.
- **ddl-auto**: Usually set to `update`, updating table structures (new columns, indices) without deleting tables at startup.
- **Table Names and Fields**: Consistent with `@Table` (e.g., `portal_event`). Fields use snake_case (e.g., `create_time`).

### 2.4 Data Initialization
- **AdminDataInitializer** (`@Order(0)`): If the `admin` table is empty at startup, it inserts the default admin username=`admin`, password=`admin123` (BCrypt), ensuring the backend is accessible.
- **DataInitializer** (`@Order(1)`): Executes when `app.init-test-data=true`; initializes only if modules are empty:
  - **initUsers()**: Registers `test@fairmontschools.com` / `test123` and sets status=`1` to bypass verification.
  - **initEvents()**：Inserts test events (titles, descriptions, activities, tasks, schedules.
  - **initSchedule()**：Inserts default schedules linked to classrooms.
  - **initClassrooms()**：Inserts classrooms with coordinates, teachers, and photos.
  - **initWelcomeVideo()**：If there is a "Latest Welcome Video" logic and it is empty, then initialize a placeholder.
- **ContentDataInitializer**: If `/Repository` did not record Content Pages, writes default Markdown for keys like academics, athletics, etc.
